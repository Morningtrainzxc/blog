# 搜索栏组件封装思考

由于在项目当中经常会遇到各种清单列表的页面开发需求，同时会伴随着各种搜索框的出现，所以将搜索框进行一个封装方便后续使用。

主要经历的思考：

  1. 如何解决高可扩展性和方便性的冲突
  2. 如何思考组件拆分粒度的设计
  3. 如何提升代码质量

# 问题一：搜索筛选栏内容不固定

​ 比如我的商机清单列表筛选项目有： 商机编码、项目名称、地州、流水号、行业等

而所有商机清单列表筛选项是：地州、区县、项目级别、时间筛选等。如何解决这个问题，我的第一想法是生成一个配置项去生成这个搜索栏。

方案一（配置项）：

​ 在父组件中去生成一个配置tableConfig，并将这个配置对象传入到子组件SearchForm中。

<template>
	<SearchForm :filter-config=tableConfig></SearchForm>
</template>
<script>
const tableConfig: Ref<Array<TableConfig>> = ref([
    {
        key: 'businessId',
        label: '商机编码',
        valueType: 'string',
        compType: 'input',
        needFilter: true
    },
    {
        key: 'projectName',
        label: '项目名称',
        valueType: 'string',
        compType: 'input',
        needFilter: true
    },
    {
        key: 'city',
        label: '地州',
        valueType: 'number',
        compType: 'select',
        needFilter: true,
        options: [
            {
                label: '成都市',
                value: 10001
            },
            {
                label: '乐山市',
                value: 10002
            }
        ],
        renderFunc: (scoped: any) => {
            return cityMap.get(scoped.city) || ''
        }
    },
    {
        key: 'processInstanceId',
        label: '流水号',
        valueType: 'number',
        compType: 'input',
        needFilter: true
    },
    {
        key: 'industry',
        label: '行业',
        valueType: 'string',
        compType: 'input',
        needFilter: false
    }
])
const cityMap = new Map([
    [10001, '成都市'],
    [10002, '乐山市']
])
</script>

在子组件SearchForm中：

<template>
    <el-form :inline="true">
        <el-form-item v-for="item in filterConfig" :key="item.key" :label="item.label" :prop="item.key">
            <!-- input -->
            <el-input v-if="item.compType === 'input'" v-model="formModel[item.key]"></el-input>
            <!-- radio -->
            <el-radio-group v-else-if="item.compType === 'radio'" v-model="formModel[item.key]">
                <el-radio v-for="opt in item.options" :key="opt.value" :label="opt.value">{{
                    opt.label
                }}</el-radio>
            </el-radio-group>
            <!-- select -->
            <el-select v-else-if="item.compType === 'select'" v-model="formModel[item.key]">
                <el-option v-for="opt in item.options" :key="opt.label" :label="opt.label"
                    :value="opt.value"></el-option>
            </el-select>
        </el-form-item>
        <el-form-item>
            <el-button @click="reset">重置</el-button>
            <el-button type="primary" @click="submit">确定</el-button>
        </el-form-item>
    </el-form>
</template>

​ 用一个v-for去变量配置对象，通过v-if以及根据这个item的compType去生成对应的筛选框。

​ 最后的效果如下：

优点：使用起来比较方便，只需要去生成配置对象就可以。以及可以用ts去约束配置对象的类型，保证配置项的健壮性。

缺点：遇到复杂的配置项，子组件内不好操作了，比如说时间，日历等组件，其需要配置的内容比较多。
